Python虚拟环境

Posted on 2021-02-19 | Post modified: 2021-03-10 | In Tools

虚拟环境作用

问题：如果在一台电脑上，想开发多个不同的项目, 需要用到同一个包的不同版本，如果使用pip命令，在同一个目录下安装或者更新，新版本会覆盖以前的版本，其它的项目就无法运行了

解决方案 : 虚拟环境

作用 : 虚拟环境可以搭建独立的python运行环境，使得单个项目的运行环境与其它项目互不影响

virtualenv

使用

conda

第一次运行conda config在用户目录生成配置文件.condarc

换源

查看源conda config --show-sources

添加清华源conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/

设置搜索时显示通道地址conda config --set show_channel_urls yes

配置环境存储位置

在配置文件中添加

删除没有使用到的包的缓存conda clean -p

创建环境并使用

创建conda create -n vf python=3.8

创建时复制某个环境conda create -n env2 --clone env1

使用activate vf，不加环境名称默认加载base

查看所有环境conda env list

包管理

用conda进行管理

查看已安装的包conda list

搜索可安装的包conda search python

安装conda install -n vf numpy，不声明环境则自动装在当前环境

conda和pip的区别：conda可以装python；有的包不支持conda；其他的包管理操作都类似
