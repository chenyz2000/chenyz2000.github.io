Vue Router

Posted on 2020-08-24 | Post modified: 2020-08-25 | In Vue

路由

后端路由：把URL地址对应到服务器上的资源，这个对应关系就叫路由

前端路由：只涉及到前端页面的跳转，不发送到后台服务器。单页面程序主要借助hash实现（URL中第一个#后面的内容）不同组件之间的切换，特点是http请求中不会包含hash的内容。通过hash切换前端页面的方式叫做前端路由

URL中的hash

安装Vue-router

先引入Vue包再引入Vue-router包，因为Vue-router要向Vue中注册一些属性

导入CDN<script src="https://unpkg.com/vue-router/dist/vue-router.js"></script>

或npm安装，模块化开发（如webpack）时，需要手动安装，用Vue.use()注册

使用

  • 导入vue-router包后，在window全局对象中，就有了路由的构造函数VueRouter，URL后面也会自动多出hash值#/

  • 创建一个路由对象

    可以向构造函数参数传递一个配置对象

    配置对象routes属性表示路由匹配规则，其值为对象的列表

    其中每个匹配规则必须有两个属性，path表示监听的路由链接地址，component表示链接对应的一个组件

    除了匹配规则外，也可以在routes中定义重定向{path:"/", rediredt:"/login"}，访问到path时自动跳转到redirect指定的链接

  • 将路由对象注册到Vue实例中

  • 在页面上放置容器

    当URL改变后，触发路由的监听事件，如果有匹配到的规则，则展示对应组件到router-view**标签所在的位置

一个URL中有多个组件

使用path-component时，每个URL地址只能对应最多一个组件，若页面中有多个router-view，则组件会在每个router-view里面显示同样的一份

当把component换成components时，可以表示一个URL地址对应多个组件

components属性的值为对象，对象中的每个元素是组件名:组件对象的键值对

然后在页面放置容器的时候

路由的嵌套

子组件的路由链接应以父组件的路由链接为前缀，但是不能在父路由的同级定义子路由

要给路由设置子路由，需要在routes数组的对象中加children属性

children属性的值是对象的数组，其中每个对象和routes数组中的对象要求相同

使用时只需要用一个router-view给父路由占位即可

若children中path改为”/login”，父子路由关系不变，访问子路由时父子路由都会展现，但是子路由访问地址变为了#/login

router-link导航标签

访问URL可以通过手动输入，也可以通过a标签跳转到路由链接，要求href="#/login"

但是在Vue Router中提供了router-link导航标签

属性to中写目标路由的连接，其中不需要写#了

属性tag，默认值为”a”，用来设置渲染到页面的标签种类，不过无论何种标签都有点击事件

属性active-class，默认值为”router-link-active”，作用是设置路由选中时，导航标签添加的CSS样式类。若想改变导航标签选中样式（如高亮），可以编辑.router-link-active样式，或在路由构造函数VueRouter中用linkActiveClass属性来全局配置默认使用的CSS类

传递路由参数

可以用组件的$route对象获取路由URL中传递的参数

$route对象有多个属性，其中fullpath表示全路径（包含参数），path表示路径（不含参数），query和params表示参数对象

通过$route.query传参

不修改routes中的path

如对于链接#/login?user=abc&passwd=123，$route对象的fullpath值为/login?user=abc&passwd=123，path值为/login，query值为{user:"abc",passwd:"123"}

通过$route.params传参

修改routes中的path，{path:"/login/:user/:name", component:{}}

如对于链接"#/login/abc/123"，$route对象的fullpath和path都为"/login/abc/123"，query为空对象，但是params为{user:"abc"}
