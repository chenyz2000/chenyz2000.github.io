Tomcat

Posted on 2020-05-28 | Post modified: 2020-05-28 | In Java Web

应用服务器

应用服务器是WEB服务器上的一个服务

它让用户可以通过ip加端口访问放在应用服务器里的Java程序

Servlet容器除了Tomcat以外，还有Jetty和Undertow

Tomcat目录

  • bin：可执行文件，startup.dat和shutdown.bat开启和关闭（Linux中是.sh后缀）
  • conf：配置文件，server.xml，可以在里面改端口号
  • lib：存放库文件，即tomcat全局依赖的jar包
  • webapps：项目文件夹，在此目录部署要运行的应用
    文件夹内需要有web-inf文件夹，其中要有web.xml、classes文件夹和lib文件夹（项目单独的class文件和jar包）
  • log：运行产生的日志文件
  • temp：运行产生的临时文件
  • work：运行产生的class文件（由JSP翻译成）

配置Tomcat

Tomcat基于Java，需要先配置Java，必须先配置Java_Home

再配置Catalina_Home，地址选择Tomcat目录

然后就可以用startup.dat启动了

如果端口号冲突（默认8080），可以在server.xml中修改

端口号若改为80即为http默认端口号，网址中可以省略端口号

之后就可以访问了

访问Tomcat

可以localhost:8080/在本地访问

默认访问webapps下的root项目

如果想访问其他项目，可以localhost:8080/examples

可以指定访问的页面，如localhost:8080/examples/index.jsp

root项目的项目名可以省略

或者页面省略时，根据项目文件夹下的web-inf/web.xml中的welcome-file-list段，按顺序访问页面

状态码：404资源不存在，403权限不足，3xx页面重定向，500服务器内部错误（代码写错）

虚拟路径

在server.xml中的host段配置

由属性appBase指定，默认是"webapps"

属性name="localhost" appBase="webapps"指本地的虚拟路径在webapps

配置其他虚拟路径

方式一（需要重启Tomcat）

在host段里加入 <Context docBase=" " path=" " />，区分大小写

docBace是实际路径，path是虚拟路径

二者都可使用绝对路径或相对于appBase的相对路径

表示访问path时实际去访问docBace

  • 例如：path为/examples，docbase为C:/examples，浏览器访问localhost:8080/examples时，自动访问C:/examples文件夹 （name+path => docBase）
方式二（不需要重启）

/conf/Catalina/localhost下创建xml文件，文件名为项目名

文件中写<Context docBase=" " path=" " />，和方式一相同的语句

虚拟主机

通过域名访问服务器时，先经过域名解析器解析成ip地址

先在本地根据host解析，解析不到再去联网解析

通过网页访问本机

配置sever.xml，在engine段添加host

engine段属性defaultHost设置为name之一

在C:/windows/System32/drivers/etc/hosts文件中加入解析

流程

  1. 输入网址
  2. host找映射关系
  3. server.xml找engine的defaulthost
  4. 通过path映射到docBase

在idea中运行Tomcat

新建项目->JavaEE->web application->next-> finish

IDEA配置Tomcat插件：

  1. 运行，编辑配置，+，tomcat server，本地，应用程序服务器配置为tomcat的安装目录
  2. 部署，+，artifact，编辑artifact，+，directory content，路径选择为webapps
  3. 当artiface为war exploded形式时，设置tomcat server的on update action为update classes and resources，点击下方面板的刷新按钮可以热部署

war包就是将当前项目打包映射到tomcat

lib中有jdk和tomcat

tomcat中用到servlet-api.jar和jsp-api.jar,也可以手动添加到buildpath

servlet的api使servlet转为JSP
src中放java程序，web中放前端页面，out中放打包的war包

WEB-INF

客户端（浏览器）访问out中的打包文件时

可以访问web中的文件

但权限不够无法直接访问web/WEB-INF中的文件，只能通过请求转发来访问

src打包后也在WEB-INF中，不能访问

=>不能直接访问servlet文件

跳转包括请求转发和重定向（重定向不可以访问web-inf）
