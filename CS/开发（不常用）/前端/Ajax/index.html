网络请求

通过Ajax、Fetch、WebSocket等API实现

Fetch

替代了Ajax

API是基于JavaScript Promise设计的

fetch是全局量window的一个方法

Ajax

Asynchronous JavaScript and XML，异步的JS和XML

同步：发完请求进行等待，获得响应后才会执行之后的代码

异步：发完请求不管响应，但会监听某个状态，当状态发生改变达到要求时，再作出相应反应

无需重新加载整个网页就可以向服务器发送请求，网页的局部数据刷新

适用场景：注册时查询用户名是否已存在，登录时提示用户名密码错误

Ajax的核心是XMLHttpRequest对象（XHR），XHR为向服务器发送请求、解析服务器响应提供了接口，使浏览器可以发出HTTP请求与接收HTTP响应、实现在页面不刷新的情况下和服务端进行数据交互

JS原生XMLHttpRequest

open()方法里面的三个参数分别是：要发送的请求类型（get、post），请求的url，是否异步发送请求

收到响应后相应数据会填充到XHR对象的四个属性：responseText，responseXML，状态码status，状态的说明statusText

onreadystatechange由readyState变更而触发

onload由请求完成时触发

XHR对象的readyState属性，表示请求响应过程的当前活动阶段，共有五个值：

  • 0：未初始化。尚未调用open()方法。
  • 1：启动。已经调用open()方法，但尚未调用send()方法。
  • 2：发送。已经调用send()方法，但尚未接收到响应。
  • 3：接收。已经接收到部分响应数据。
  • 4：完成。已经接收到全部响应数据，而且已经可以在客户端使用了。

发送消息

GET 方法发送信息，可以向 URL 末尾添加字符串参数

POST 请求，对象则拼接成字符串形式

jQuery Ajax

通过callback+XHR实现

语法：$.ajax({name:value, name:value})

其中键值对可以为：

  • url: string，发送请求的地址（默认为当前页）
  • data: string或object，发送的数据
  • type: string，请求的方式（默认为"get"）
  • async: boolean，是否异步处理（默认为true）
  • contentType: string，发送内容的编码类型（默认为"application/x-www-form-urlencoded"）
  • dataType: string，预期返回的数据类型
  • username: string和password: string，响应HTTP访问认证请求的用户名和密码
  • beforSend: function，发送请求前对XHR对象进行操作，XMLHttpRequest为唯一参数
  • complete: function，请求完成后调用，参数为XMLHttpRequest对象和成功信息textStatus
  • success: function，请求成功后调用，参数为返回并经过处理的数据data和textStatus
  • error: function，请求失败后调用，参数为XMLHttpRequest对象、错误信息textStatus、捕获的错误对象errorThrown(可选)
