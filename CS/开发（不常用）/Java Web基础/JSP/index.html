JSP

Java Server Pages

JSP执行流程

第一次访问

  1. 客户端提交请求
  2. tomcat服务端（JSP引擎）将JSP翻译成Java源文件（即Servlet），从tomcat/work中找Java文件能找到包含Servlet的_jspInit、 _jspService方法和out.write()的
  3. 并将其编译为.class文件，执行.class文件返回结果

第二次访问

  1. 直接访问class文件
  2. 若服务端代码修改，则按第一次访问进行

统一编码

  • jsp翻译到Java的编码：.jsp文件中的contentType和pageEconding属性
  • 浏览器读取时的编码：.jsp文件中的content属性
  • 文本文件的编码
    一般都用utf-8

Java运行Tomcat时，会复制一份配置文件

将web工程以包的形式上传到服务器，而不是部署到webapps中

JSP语法

基本格式

指令

  • <%@ page attr:"val" %>，定义整个JSP页面的相关属性，其中attr有
    • language，指定JSP页面编写的脚本语言，一般为"Java"
    • import，导入的类，如import="java.util.*,java.sql.*"
    • pageEncoding，为JSP文件编码
    • contentType，为浏览器解析编码
  • <%@ include attr:"val" %>，重用代码
    • 只有file一个属性，类似于c中的include，相当于直接把代码拷贝进来
  • <%@ taglib attr:"val"%>，需要JSLT的核心库进行配置
    • uri

脚本Scriptlet

脚本转为HTML源代码body中的内容

  • <% 局部变量，常规Java语句 %>
  • <%! 全局变量、定义方法 %>，可以被其他段的语句调用
  • <%= 输出的字符串或值 %>，相当于<% out.print(字符串) %>

多段脚本按照在整个JSP文件中的顺序进行，可以互相调用

在JSP脚本中println也不换行，换行需要<br>

可以解析字符串中的HTML代码

注释

  • HTML注释：<!-- -->，可以在源代码中看到
  • Java注释
  • JSP注释：<%-- --%>

一般而言，修改配置文件需要重启Tomcat，修改页面文件不需要重启

JSP九大内置对象

自带的，不需要new也能使用的对象

  • out：向网页页面输出

  • request：网页向服务端的请求信息

    response：服务器向网页的响应信息

  • session：表示会话，保存用户的信息

    application：所有用户的共享信息

  • pageContext：表示当前JSP页面的运行环境

  • page, config, exception

request

  • .getParameter(name)，根据表单的name获取value

    .getParameterValues(name)，返回数组

  • .setCharacterEncoding(“utf-8”)，设置请求时的编码

    （Tomcat7以前默认为ISO8859-1，Tomcat8以后为utf-8）

  • .getRequestDispacher(“跳转向的页面”).forward()，请求转发

  • .getServletContext()

response

  • .sendRedirect(“url”)，重新定位到页面
  • .setHeader(attr,value)
  • .addCookie(Cookie c)

session

  • .getAttribute(“name”)，返回对象
  • .setAttribute(“name”, Object value)

application

pageContext

  • .getOut()和.getRequest()等等，可以获得当前页面的其他八个对象
  • .

cookie

存储在用户浏览器上的数据，用来辨别用户身份

request注册

register.jsp和show.jsp

在register.jsp中的form表单属性写action="show.jsp"
