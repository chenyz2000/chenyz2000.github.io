静态资源

网页中常用的静态资源

  • CSS类文件（.css .less .sass .scss）
  • JS类文件（.js .jsx .coffee .ts）
  • 图片
  • 字体文件（.ttf .svg .eot .woff）
  • 模板文件（.vue .ejs .jade）

网页静态资源多会产生的问题

  • 要发起很多的二次请求，网页加载速度慢
  • 要处理复杂的包之间的依赖关系

解决方案

  • 合并、压缩、精灵图、图片转Base64编码 ==> 解决二次请求
  • 使用webpack或requireJS等 ==> 解决包之间的依赖关系

webpack

基于Node.js开发的前端项目构建工具，需要先安装Node再安装webpack

Gulp是基于task任务进行构建的，webpack是基于整个项目进行构建的

安装

全局安装：npm i webpack -g

本地安装：项目根目录运行npm i webpack --save-dev安装到项目开发依赖中

本地安装的不能在终端中用命令执行，可以在package.json的scripts中配置命令，如"scripts":{"dev":"webpack-dev-server"}，然后可以用npm run dev运行此命令

目录

dist：发布完成后的产品级内容

src：包含文件夹css、js、images等，包含文件：项目首页index.html、js入口文件main.js

npm init初始化项目，获得node_modules文件夹和package.json文件

node_modules类似maven仓库一样，存放npm install的工具

webpack打包处理JS

不要在index.html里面用script或link导入包和CSS，不然会发送二次请求

而是在main.js中使用import xxx from "yyy"，这是ES6+导入其他模块的方式（相当于Node中的const xxx = require("yyy")）

然后只要在index.html中导入一个main.js就可以了

有的浏览器不支持ES6，导入main.js不能正常运行

在项目根目录运行webpack .\src\main.js .\dist\bundle.js，将main.js文件进行webpack打包处理生成bundle.js文件（webpack可以处理一部分ES6语法）

再在index.html中导入bundle.js就可以解决了

可以看出，webpack可以处理JS文件之间的依赖关系，可以把高级的浏览器不识别的语法转为可识别语法、解决兼容问题

配置webpack输入和输出

在项目根目录创建webpack配置文件webpack.config.js

可以配置打包的输入文件和输出文件

配置好之后用命令webpack就可以执行打包

执行过程为：发现命令中没有指定参数->在项目根目录中查找配置文件->解析执行配置文件，得到配置对象->从配置对象中获得输入文件和输出文件->进行打包

自动打包编译

webpack相对于webpack-dev-server，等于node相对于nodemon

本地安装webpack-dev-server工具，其依赖于本地安装的webpack

在package.json中配置运行命令webpack-dev-server即可用npm运行。

它提供了几个参数：

  • --open自动打开浏览器
  • --port 8888修改端口
  • --contentBase src修改服务器的根目录
  • --hot模块热替换。功能一是只局部更新，而不重新生成bundle.js；功能二是对于css的更改，浏览器访问页面时不需要刷新（js更改需要刷新）

配置参数的方式：

  • package.json中，在webpack-dev-server命令后直接加

  • webpack.config.js中（太麻烦了，仅供了解）

运行时，webpack-dev-server会把项目自动托管到服务器上，因为它是一个小型的node.js Express静态文件服务器

它会自动创建自己的bundle.js，并将其托管到项目根目录（需要修改index.js中的导入为根目录的bundle.js），但是此文件是存放在内存中（更快）而不是在硬盘上

html-webpack-plugin

插件的作用：根据模板html在内存中生成html，并在其中自动引用打包好的bundle.js

在webpack.config.js中进行配置plugins

webpack打包处理CSS

webpack默认只能打包处理JS类的文件，CSS需要第三方Loader加载器style-loader和css-loader

在main.js中导入CSS文件import "./css/index.css"

main.js中如果要通过路径的方式引入node_modules文件夹中的文件，可以直接写包的名称+文件路径，如import "bootstrap/dist/css/bootstrap.css"，默认会从node_modules中找

并在webpack.config.js中进行配置module对象的rules数组

调用过程：先校验文件的类型，若为JS则直接打包，若不为JS则去配置文件module.rules中找匹配规则，若能找到则根据规则进行处理。处理时调用的Loader顺序是从右到左，一个Loader处理完之后把结果传给下一个Loader，最后把结果传给webpack

若为.less文件，则规则是style-loader<=css-loader<=less-loader，此外要安装一个less-loader的内部依赖less不过不用写在规则里

若为.scss文件，则规则是style-loader<=css-loader<=sass-loader，此外要安装一个sass-loader的内部依赖sass不过不用写在规则里

  • limit。webpack将小于limit指定字节数的图片转换成Base64字符串，节省了图片的二次请求；大于等于字节数的图片不转为Base64字符串，是原来的图片资源
