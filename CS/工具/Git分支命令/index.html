Git分支命令

Posted on 2020-07-13 | Post modified: 2023-04-29 | In 工具

分支的含义

实质上，分支、远程分支和HEAD都是指向commit的指针

commit的哈希值、分支的名字、两种作用下的HEAD ，都可以作为命令中用到的参数

HEAD

HEAD既可以作为指向branch的指针，即HEAD→main→c1

HEAD也可以作为指向commit的指针，即，即HEAD→c1

查看分支

git branch查看本地分支

-r参数查看远程分支

-a参数查看本地和远程分支

删除分支

git branch -d <name>删除本地分支

此外还有删除远程分支、删除本地对远程分支的跟踪等命令

创建分支

创建新分支

git branch <newBranch>

创建新分支并切换

git checkout -b <newBranch> [<Node>]

Node作为新分支的模版，可选

相当于git branch <newBranch>; git checkout <newBranch>

切换checkout

git checkout <commit/branch>改变HEAD的指向

移动HEAD到某个结点git checkout <Node>

移动分支到某个结点git branch -f <Branch> <Node>

相对引用

git checkout <commit/branch/HEAD><ralative>

relative为^或~表示前继结点，relative为~3表示上溯3个结点，这样可以不用输入哈希值

切换时遇到The following untracked working tree files would be overwritten by checkout

解决方法一：git rm –cached

解决方法二：git clean -d -fx

合并merge

当前分支A指向结点a，待合并分支B指向结点b

将待合并分支B并入当前分支A，使用命令：git checkout A; git merge B

  1. 若a和b无继承关系，则新结点c继承a和b，当前分支A指向结点c
    如下图，main指向c3，执行git checkout main; git merge bugFix获得如下的结果
  2. 若当前分支结点a是待合并分支结点b的父结点，则A指向到较新的结点b
  3. 若当前分支结点a是待合并分支结点b的子节点，则A比B已经较新，不改变指向

继承关系的父子结点中，父结点表示旧的提交，子结点表示新的提交

变基rebase

分支A指向结点a，目标分支B指向结点b

将分支A的修改汇入目标分支B，使用命令：

git checkout A; git rebase B

或git rebase B A

  1. 若二者无继承关系，则新结点c继承b，A指向c，原A分支隐藏
  2. 若二者有继承关系，则使用rebase和merge道理相似。A要向B汇入；若A是B的父节点，则A更新指向到B；若A是B的子节点，则A已比B新，不更新指向

rebase原理

  1. 找到当前分支和目标分支的最近共同祖先
  2. 对比当前分支相对于该共同祖先的历次提交
  3. 提取相应的修改并存为临时文件
  4. 将当前分支指向目标分支
  5. 将之前临时文件的修改依序应用

其中，1好像可以通过rebase的–onto进行设置，但是还没特别整明白

交互式变基

git rebase -i可以进行交互式变基，可以选择结点、调整顺序、调整信息等，不像原来的rebase只能默认全部变基

所以可以实现子结点向父结点变基，不会显示A比B新

使用技巧：git rebase -i可以将本地的多次提交合并为一个，以简化提交历史

merge对比rebase

merge和rebase是git中整合来自不同分支修改的主要方法

原有分支如下

merge操作如下，把当前分支和公共分支的commit合并起来，形成一个新的commit，会保留被合并的分支的记录

rebase操作如下，把当前分支的commit放到公共分支的最后面，就好像从公共分支又重新拉出来这个分支一样，所以叫变基。不保留被合并的分支的记录，rebase会让历史更简洁

下游分支更新上游分支内容的时候使用 rebase

上游分支合并下游分支内容的时候使用merge

cherry-pick

git cherry-pick <Node1> <Node2> <Noden>

意思是择优挑选，将一些提交复制到当前位置后，按照Node1、Node2…Noden的顺序，HEAD←Node1←Node2←Noden，依次为子节点

应用举例：如何对已提交的代码进行修改

问题：对于提交c1←c2←c3←c4←c5(HEAD)，如果想要修改c3怎么办？

如果使用reset，则c4和c5也会被回退

此时使用cherry-pick

  1. 用cherry-pick或rebase -i将原分支中待修改的提交c3，调整为修改分支temp中的最新结点c3’
  2. 在修改分支temp中对c3’进行修改，之后用commit --amend对c3’进行提交获得c3’’
  3. 将c3’'和原分支中c3以外的结点按照原来的顺序放入修改后的分支

这一过程涉及了至少三条分支：原分支、temp修改分支、修改后的分支
