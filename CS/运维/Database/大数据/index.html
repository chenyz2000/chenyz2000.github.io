大数据

Posted on 2021-04-19 | Post modified: 2021-05-23 | In Database

参考：小白学习大数据

概述

问题一 存储

2000年，谷歌意识到数据大量增长，从TB增长到PB，单机难以存储

于是谷歌提出分布式文件系统(DFS)

把文件分成多个文件块，并且每个文件块可以放在若干个机器上有多个备份，实现容灾

使用者直接使用DFS的接口，不需要关注文件块的底层存储和查找细节

问题二 计算

MapReduce，Map指把任务进行分发，Reduce指把计算的结果汇总

Hadoop

安装

基于Java

安装后的配置详见官网，要修改core-site.xml和hdfs-site.xml等

在Hadoop所在的文件夹中，运行的命令位于./sbin/start-dfs.sh，启动hdfs（Hadoop的dfs）文件系统

使用./bin/hadoop fs -xxx就可以对hdfs文件系统进行操作，其中xxx是命令，如ls、mkdir等

概念

存储的若干台机器称为DataNode

调度管理者称为NameNode，数据存储的位置保存于其中

SecondaryNameNode，作为NameNode的备份，会和NameNode进行同步

MapReduce

一段Java程序

需要Hadoop core、Apache Hadoop common、Apache Hadoop MapReduce Core依赖

把程序打成jar包，可以不把依赖的jar包打进来因为Hadoop下面有jar包（可能版本不一样）

运行命令./bin/hadoop jar demo.jar 主类全类名 输入目录 输出目录

要求输出目录一定是空目录或不存在

MapReduce在传输的时候使用文件的形式进行传输，涉及到大量数据的序列化和反序列化，Hadoop对这种IO进行了特殊的优化

org.apache.hadoop.io包中提供的基础类型：

  • Text类，对应Java中的String
  • IntWritable类，对应Java中的int
  • LongWritable类，对应Java中的long

Hive数据库

基于Hadoop的一个数据仓库工具，用来进行数据提取、转化、加载，这是一种可以存储、查询和分析存储在Hadoop中的大规模数据的机制。

hive需要数据库记录信息，能将结构化的数据文件映射为一张数据库表

hive提供了SQL查询功能，因为文件存放在hdfs中，所以运行SQL语句就转变成MapReduce任务来执行，是MapReduce操作SQL化的封装，不必开发专门的MapReduce程序。

默认端口为10000，用beeline（hive命令行客户端）连接

beeline -u jdbc:hive2://localhost:10000

连接成功后，就是一个类似mysql命令行的界面

如show tables;，select，create

hive的sql成为hql，和sql相比，不支持事务，但支持sql操作封装成函数，有很多对大数据的函数

其他大数据框架

Hadoop家族

Hadoop

Hive

Hbase：基于Hadoop的K-V型数据库

Spark家族

和Hadoop比，中间结果用内存存储，Hadoop更像是离线的分析

SparkRDD：分布式弹性数据集，给数据写好有向路径，没有路径的数据是惰性的

SparkStream：流式数据分析，在线，基于时间窗口（一簇）

Storm：实时，比SparkStream更实时

PostgreSql家族

Pipelinedb：使用sql的流式分析数据库

GreenPlum：支持事务
