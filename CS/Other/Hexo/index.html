环境准备

git和Nodejs

Nodejs会自带安装npm，并默认将其加入环境变量

hexo

命令npm install -g hexo-cli，可以加--prefix 路径设置安装路径，否则是默认路径

遇到warn警告可以不管

之后把hexo.cmd所在的路径加入环境变量，应该是直接在安装路径中而不是在bin中

创建一个hexo博客项目

创建博客项目

命令hexo init 本地博客目录

安装依赖文件

在本地博客目录，命令npm install

新建博文

在本地博客目录，命令hexo new “test”

会在sources/_post/下生成test.md

新建页面

在本地博客目录，命令hexo new page categories

会生成sources/categories/index.md

更新html

每次修改，都需要在本地博客目录下使用如下命令更新html

generate是将md解析成html

在本地博客目录，命令hexo server或简写hexo s

并且修改_config.yml的theme为主题名称

博客配置

Trouble：本地server打开主题与github托管的项目主题不一致，github不能正确打开主题。

Solution：修改本地博客的配置文件，将url改为https://chenyz2000.github.io/

报错：cannot get /categories/20%。20%是空格，在next的配置文件中，需要把categories等的路径后的空格去掉

文章的tags:和categories:后一定要加一个空格

Trouble：报错YAML Syntax Error Bad indentation of a mapping entry。

Solution：yml配置文件语法有缩进，注意是不是哪里缩进出错

Trouble：部署时，github部署失败，发送邮件the page build failed for the master branch，邮件中提到Jekyll。

Solution：这个问题是github自己抽风，大概是页面上存在一些Jekyll不支持的字符，但是我搭建的博客不需要使用Jekyll，解决方法是，在提交github的仓库里，建立名为.nojekyll的空文件，之后重新部署。

更改永久链接：修改站点配置的permalink，默认值是:year/:month/:day/:title/。

categories、tags下的index.md中加内容没有效果，只有about中有效果
