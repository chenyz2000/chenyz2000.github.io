npm

NPM，node package manager，是一个Node.js的包管理器（类似linux的apt）

我们在传统开发的时候，jQuery.js大多都是百度搜索，然后去官网下载，或者直接引入CDN资源，这种方法太过于麻烦。如果以后遇到其他的包，这个包的代码本身可能还调用了其他的包（即存在依赖关系），那么我们要在自己的项目中引入一个包将变得十分困难。现在我们有了NPM这个包管理器，直接可以通过npm install xxx包名称的方式引入它，比如说npm install vue

作为一个前端开发，即使你不用Node.js写独立程序，也得配一个Node.js运行环境，毕竟很多前端工具都是使用它写的。

常用命令

全局安装和本地安装

只有本地安装才可以用require()的方式引入模块

全局安装的包可以直接在命令行中使用

本地的优点是避免包版本冲突，缺点是会重复安装

建议不提供命令行的包使用本地安装（如flv.js），经常用到命令行的可以全局安装

创建项目

在一个空文件夹下npm init，生成package.json

package.json中包括项目描述、项目依赖的包、插件配置等信息

一个包含package.json的文件夹被视为一个项目

然后在此项目中安装软件包npm install xxx --save

其中--save的作用是把这个软件包依赖信息写入package.json中的devDependencies

（或–save-dev表示开发环境）

安装的软件包放在项目中的node_modules文件夹里

根据package.json中的依赖信息，运行npm install，即可下载所有的包

哪怕删掉node_modules也没事，只要保证package.json即可

简写命令

在package.json的scripts中规定简写的命令

如默认的"test": "echo \"Error: no test specified\" && exit 1"

可以通过npm run test运行
