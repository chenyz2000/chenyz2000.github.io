JSP和Servlet

Posted on 2020-12-21 | Post modified: 2020-12-21 | In Java Web

Servlet和JSP都是SUN公司提出的解决服务器端的操作，Servlet早于JSP，JSP会转换成Servlet

JSP比较简略，早期没有JSP时，就用Java中的servlet接口

Servlet是一套用来处理网络请求的Java Web规范，JSP是建立在Servlet规范之上的Web开发技术

Servlet形式是Java接口（狭义）、Java程序（广义），JSP形式是在HTML中嵌套Java代码

Servlet和JSP都应用于服务器端（部署于Tomcat服务器），用于与数据库通信、动态生成网页

Servlet运行在Servlet容器（Tomcat，应用服务器）中

web服务器有Tomcat，Resin，Jboss，WebSphere，WebLogic

JSP在Servlet容器中由JSP引擎转化为Servlet

JSP会转化为Servlet（即翻译成Java代码），编译为class文件后执行

Tomcat常用作 servlet容器，在 Java Web应用开发中广泛使用

当然，Tomcat也包括了web服务器的功能，也可提供 HTML 页面服务

因此tomcat是web服务器的扩展,也可以理解为tomcat就是web服务器

为什么说tomcat是servlet容器呢？字面意思容器就是可以容纳很多东西，众所周知，在tomcat上部署应用程序的时候，我们是把应用程序放在tomcat根路径下的webapps文件夹下,而这个文件夹下可以放置多个应用程序,我们可以通过请求来决定具体访问那个应用

tomcat的整个运行流程：

  1. 用户通过浏览器向服务器发送请求
  2. tomcat接收请求后解析请求具体访问哪个应用
  3. tomcat创建一个HttpServletRequest对象,将用户发送的请求封装到这个对象里
  4. tomcat创建一个HttpServletResponse对象
  5. Servlet容器调用HttpServlet对象的service方法，把HttpRequest对象与HttpResponse对象作为参数传给 HttpServlet对象
  6. HttpServlet调用HttpRequest对象的有关方法，获取Http请求信息
  7. HttpServlet调用HttpResponse对象的有关方法，生成响应数据
  8. Servlet容器把HttpServlet的响应数据结果传给浏览器,浏览器再根据返回的response进行相应的渲染
