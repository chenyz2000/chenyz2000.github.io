Docker

Posted on 2020-10-27 | Post modified: 2020-10-27 | In 微服务与云原生

原来的application，一个应用就要配置一个mysql、redis等服务，开发的时候要配置一套，部署的时候要配置一套，难以确保完全一致，并且浪费时间。

Docker是一个容器技术，容器中装的是应用服务。

Docker可以部署MySQL，NGINX，Tomcat，jdk，Python，Redis，MongoDB，RabbitMQ，elasticsearch，Jenkins，Linux等项目环境

使用Docker能保证不同机器上的环境高度一致。

Docker和虚拟机做类比

镜像类比虚拟机的iso文件

容器类比正在运行的虚拟机

tar文件类比vm的vm的vmdk文件

Dockerfile是一个配置文件，通过记录”如何构建“的步骤，来构建一个镜像

命令

docker images 或者docker image ls

docker pull nginx

docker run -d -p 80:80 镜像名，-d为后台运行不阻塞指令窗口，-p指定内外端口映射（对外：内）

docker ps查询所有正在运行的容器

docker exec -it 容器ID值 bash，进入容器内部进行修改

docker rm -f 容器ID值

docker rmi -f 镜像ID值

docker commit 容器ID值 镜像名

Debian一句话安装docker

见https://www.runoob.com/docker/debian-docker-install.html

镜像源

排错

Got permission denied while trying to connect to the Docker daemon socket at unix:///var/run/docker.sock: Get http://%2Fvar%2Frun%2Fdocker.sock/_ping: dial unix /var/run/docker.sock: connect: permission denied

是因为权限不足，没有用root用户使用docker

docker run -d --network=host --env PORT=9000 --restart=always --name portainer -v /var/run/docker.sock:/var/run/docker.sock -v /usr/dockerfile/portainer/data:/data docker.io/portainer/portainer

arm架构的docker，网络只有host模式（x86下默认是bridge），需要把-p 9000:9000换成--network=host --env PORT=9000

host表示占据宿主机的端口，bridge表示有自己虚拟出来的端口

filebrowser比较复杂，想要把其中的配置文件.filebrowser.json和database.db映射出来，需要更多步骤。因为docker -v一般用来映射目录，它也可以映射文件，但是需要宿主机有该文件，不然docker会在宿主机上自动创建文件夹。所以想要映射filebrowser中的文件，可以先运行一次镜像，此时虚拟机里已经有了两个文件，可以在宿主机用docker cp，把虚拟机里的文件拷贝到宿主机来。之后，就可以重新创建一个镜像了。
