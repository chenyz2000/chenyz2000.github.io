容器编排技术

Kubernetes是一个开源容器编排引擎和容器集群管理工具，用来对容器化应用进行自动化部署、扩缩和管理

可以提供的功能：

  • 服务发现和负载均衡
  • 存储编排
  • 自动部署和回滚
  • 自动装箱计算

Kubernetes是云原生托管的第一个开源项目，能够运行在Kubernetes之上的应用被称为云原生应用

在Kubernetes之前，Spring Cloud是云原生的探路者，两者有很多功能是重合的，包括：服务注册和发现，API网关，负载均衡，配置管理，但是Spring Cloud只能用于Java开发，Kubernetes是语言无关的

虚拟机和容器的区别：容器非常轻量，只有几十兆大小，它实现了进程级的隔离，不同容器还是要共享主机的内核；每个虚拟机则包含一个独立的操作系统，会占用大量的系统资源。容器非常适合用于微服务架构中。

k3s是Kubernetes的轻量的发行版

Kubernete的作用：

  • Pod：Kubernetes中的一个部署单元，具有单个Internet协议地址
  • 水平缩放：Kubernetes中的重要功能，此功能使用HorizontalPodAutoscalar根据观察到的CPU利用率自动增加或减少deployment、replica Controller、replicaset或Statefulset中的pod数
  • 自动装箱：Kubernetes帮助用户声明其容器的计算机的最大和最小资源
  • 服务发现和负载平衡：Kubernetes为一组容器分配IP地址和DNS名称, 并在它们之间平衡负载
  • 自动部署和回滚：Kubernetes使用部署, 将更改和更新分发到应用程序或其配置。如果系统中发生任何问题, 则此技术会立即为你回滚这些更改
  • 持久存储：Kubernetes提供了一项称为“持久存储”的基本功能来存储数据, 在pod被终止或重新安排之后不会丢失
  • 自我修复：此功能在Kubernetes的概念中起着重要的作用。在执行过程中失败的那些容器, Kubernetes会自动重新启动它们。并且, 那些不回复用户定义的运行状况检查的容器将阻止它们自动运行

Kubernetes架构

一个Kubernetes集群包含一个master，以及多个node，属于主从模型

每个Node对应一个服务器（实体或虚拟机），master可以和Node安装在同一台机器（但是建议master单独部署）

Master（control plane）

负责管理node和维护集群状态，负责所有任务的分配，但是不跑任务

1、API Server

整个系统的对外接口，供客户端和其它组件调用

2、Scheduler

负责对集群内部的资源进行调度，Scheduler监视新创建没有分配到Node的Pod，为Pod选择一个Node

3、Controller manager

对控制器Controller进行管理，Controller包括

  • Node Controller节点控制器
  • Replication Controller副本控制器：负责维护系统中每个副本中的pod
  • Endpoints Controller端点控制器：填充Endpoints对象（即连接Services＆Pods）
  • Service Account Controller服务账号控制器和Token Controller：为新的Namespace创建默认帐户访问API Token

4、etcd

K8S自带的一个键值存储数据库，存储配置数据和集群状态信息

Node（worker，compute machine）

负责执行由master分配的请求任务，运行实际的应用和工作负载

1、kubelete

每个node都包含一个kubelete，会定期向master汇报自己Node的状态（Node状态、Pod状态、容器健康检查），并接受来自Master的指示采取调整措施

2、Kube-Proxy

Kube-Proxy负责Node在K8S的网络通讯、以及对外部网络流量的负载均衡

3、Container Runtime

  • docker
  • kvm2
  • virtualbox
  • rkt

Pod

每个node上运行若干个pod，pod由若干个容器组成，可以被称为容器集

pod是Kubernetes的最小调度单位，每个pod对应一个进程

每个pod有一个ip，其中的容器共享同个ip

在pod中可以设定若干个volume共享存储，其中的容器可以使用这个共享存储

请求的传递：Ingress → Service → Pod

Service可以看作一组提供相同服务的Pod的对外访问接口，Service可以跨Node
