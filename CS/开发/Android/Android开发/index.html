IDEA中右键→new→Vector asset，有很多矢量图标可以选择

160dp≈1英寸，可能会有偏差

  • material design：样式设计
  • Jetpack：组件库，适合大型、有复用需求的项目
  • Flutter：跨平台UI包，需要学习一门新语言 Dart
  • RxJava：异步操作、多线程库
  • Retrofit：网络请求框架
  • Glide：图片加载框架
  • Realm：替代SQLite的更轻量的数据库
  • MVP设计模式，Dagger2依赖注入

app开发

  • 原生：安卓用Java/Kotlin，IOS用Object-C/Swift

  • 混合：Html+容器（PhoneGap/Cordova/Ionic）

  • 跨平台——编译：前端JS+前端框架+自研引擎。复用前端技术，更易开发。

    先用特定语言编写，编译时转换为原生语法

    • React Native：脸书家，使用JS作为编程语言，生态好，最稳妥，跨安卓+IOS。eg：抖音、京东、网易云音乐、QQ音乐、百度、小红书、爱奇艺、喜马拉雅
    • weex：阿里家，基于Vue，跨安卓+IOS+网页。eg：淘宝、钉钉

  • 跨平台-编译2：自研语言+框架+引擎。性能好

    • Flutter：谷歌家，Dart语言，性能好，跨安卓+IOS。eg：QQ、快手、闲鱼、美团
    • Xamarin：微软家，C#语言
    • QT：QML语言

  • 跨平台-小程序：

    • uniapp：基于Vue，相当于Vue+小程序，可跨所有平台+小程序
    • taro：基于React
    • chameleon

原生用来调用硬件，UI用前端技术开发，二者之间用webview的js桥连接

推荐：公司开发，最稳妥用RN，Flutter的比例在接近RN；需要小程序，uniapp适合快速开发、小型项目

Android开发简介

开发工具：Android Studio

Android Studio使用Gradle构建项目

需要JDK + Android SDK

Android最低支持版本选则5.0就好

安卓自身系统库包含：媒体库支持播放音视频、web浏览器引擎、轻量级数据库SQLite、2D库、3D库、矢量渲染库

文件目录

  • app/src/main/
    • java/<包名>：
      • MainActivity.java：窗口的操作
    • res/：资源文件
      • drawable/：放图片或小型布局的xml
      • layout/：放窗口结构的布局文件
      • values/：存放定义的值
      • mipmap/：位图
    • assets/：不编译的资源文件，如html
    • AndroidManifest.xml：配置文件，注册用到的所有activity界面
  • app/src/build.gradle：配置客户端版本号和依赖的包

@引用

res中的内容，包括layout、value和drawable等文件夹，可以在xml里和java里引用

建议使用需要数值的时候进行引用而不是写具体数值

值得一提的是，values/styles.xml可以设置多个属性合并为一个风格，根据风格名引用

使用建议

  • 在布局简单的情况下，优先使用LinearLayout，其次考虑RelativeLayout和ConstraintLayout。简单布局下，LinearLayout代码量相对较少
  • 在布局复杂的情况下，优先使用RelativeLayout和ConstraintLayout，不建议使用LinearLayout各种嵌套，过多层次的嵌套会增加绘制时间
  • 如果习惯使用RelativeLayout，则不必刻意追求使用ConstraintLayout
