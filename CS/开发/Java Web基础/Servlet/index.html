Servlet程序

一个Servlet程序的创建通常是继承了HttpServlet类，并重写对HTTP请求的响应处理的方法

实现类HTTPServlet继承抽象类GenericServlet，GenericServlet实现接口Servlet

方法有doGet()、doPost()、doPut()、doDelete()、service()、init()、destroy()等等，主要使用的是doGet、doPost，它们两个方法经常一个调用另一个

现在分析doGet()方法

两个参数分别是HttpServletRequest和HttpServletResponse类的对象

在JSP中，内置对象request和response就是这两个类型的

Servlet中进行输出response.getWriter().append()，与JSP的out对象不同，out带缓存机制

response.getWriter().append()不带缓存，类似StringBuilder

response.getWriter()也有println()方法

Servlet想要外部访问需要进行配置，配置Servlet名、完整路径、url访问地址、加载顺序、初始参数

方法一 web.xml文件配置

servlet标签用来定义servlet，其中servlet-name为自定义的标识名，servlet-class为Servlet类的全名

servlet-mapping标签用来映射url到servlet，其中servlet-name使用之前定义的标识名，url-pattern为映射的url

方法二 注解配置

访问url需要注意网站根目录，根目录可以用request.getContextPath()得到

在form表单的action中写"/hello"就是一种访问方式

Servlet

Server Applet，是用Java编写的服务器端程序。

Java Applet是指用Java编写的能够在Web页中运行的 应用程序，它的可执行代码为class文件。

Servlet本身是一组接口，在javax.servlet.包里

java和javax有什么区别？都是官方jdk，java是基础的包，javax是扩展的包，如util和lang在java中，servlet和sql在javax中

广义的Servlet指任何实现了这个 Servlet 接口的类

一个类实现servlet接口，则它就具备了接收客户端请求和做出响应的功能

使用

servlet接口的方法

  • init初始化

  • getServletConfig

  • service服务，是整个的核心，每次进入页面都通过它收到请求并进行响应

    两个参数，分别是请求对象sevletRequest和响应对象sevletRespons

    • 如sevletRequest.getParameter("key")可以获得传入的参数。(传入的参数在地址后的?参数=值中)
    • sevletResponse.getWriter()就可以进行对响应进行写入，即可以在浏览器看到。（可以用setContentType("charset=utf-8")设置响应的编码）

  • getServletInfo

  • destroy销毁

Servlet生命周期

init，service，destroy三个方法属于生命周期

每次访问，检查servlet容器里是否有servlet对象，若没有则创建对象并init，之后对对象进行service调用
当服务器关闭，其中的对象都释放，调用destroy
对象的创建通过tomcat，使用反射机制，自动调用无参构造方法创建
