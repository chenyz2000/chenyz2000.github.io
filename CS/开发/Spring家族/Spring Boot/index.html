Spring Boot

官方文档

简化Spring，将其中复杂的XML配置Bean的部分砍掉，是Spring的再封装

核心思想：CoC约定大于配置，Spring Boot已经配置好了约定俗成的部分，开发人员只需要定制不符合约定的部分

Spring Boot使得易于集成，如集成数据库、缓存、权限、web容器、监控、消息队列

原本开发Web项目需要：配置maven依赖，配置web.xml加载Spring，配置数据库连接配置Spring事务，配置加载配置文件的读取开启注解，配置日志文件，…，部署Tomcat进行调试

而使用Spring Boot创建即可使用：建立一个Spring Boot工程，配置maven依赖，接可以直接启动

Spring Boot配合Spring Cloud：Spring Boot用于开发微服务中的一个个服务，SpringCloud用于管理Spring Boot开发的多个服务

创建项目

通过maven项目构建

  1. 创建maven项目

  2. pom.mxl中导入Spring Boot相关依赖

    其中指定的版本就是Spring Boot的版本

  3. 加入需要的starter

  4. 写一个主程序，启动Spring Boot

    在启动类直接标注@SpringBootApplication相当于

  5. 编写Controller、Service等

  6. 测试

    启动，自动在8080端口启动了Tomcat

    访问localhost:8080/hello

  7. 部署

    导入插件，可以将引用打包成可执行的jar包

    之后用maven的package打包，打完包后放在项目文件夹的target目录中

  • [ ] 此处打包失败，尚未解决

    之后可以直接用java -jar运行jar包

    jar包中，BOOT-INF/classes是程序的字节码，BOOT-INF/lib是程序依赖的jar包（包括tomcat-embeded）

Spring-boot-starter指Spring Boot场景启动器

Spring Boot将所有的功能场景都抽取出来，做成一个个Starters（启动器），只要在项目里引入启动器，相关场景的所有依赖都会导入进来

自动配置

@SpringBootApplication注解表示SpringBoot的主配置类，SpringBoot应该运行这个类的main方法来启动SpringBoot应用。使用这个注解能实现自动扫描包和自动配置默认配置的功能。

包含以下三个子注解：

  • @SpringBootConfiguration 配置，和其子注解@Configuration含义一样，只是在Spring Boot中重新定义了一下
    @Configuration是Spring的IoC容器配置类，表示该类是bean定义的源，即其中定义了bean

  • ==@EnableAutoConfiguration 开启自动配置==
    包括以下两个子注解，重要的是@Import()

    • @AutoConfigurationPackage
      注册了一个bean，用来把扫描路径保存到一个全局变量中

    • @Import({EnableAutoConfigurationImportSelector.class}) ==自动装配的核心==
      获取所有类的全名，加载到IoC容器中；@Configuration类下面被@Bean修饰的也会被加载为Bean
      将所有符合自动配置条件的Configuration配置加载到 IoC 容器，选择和加载配置，与@EnableScheduling、@EnableCaching、@EnableMBeanExport 等类似

      nableAutoConfigurationImportSelector的父类AutoConfigurationImportSelector的方法selectImports()中借助了Spring框架原有的一个工具类SpringFactoriesLoader，得以完成此自动配置

      SpringFactoriesLoader主要功能就是从classpath中搜寻所有META-INF/spring.factories 配置文件，将扫描到的内容包装成properties对象，格式为Key=value。之后根据EnableAutoConfiguration作为 Key，获取对应的一组自动配置类(xxxAutoConfiguration)，汇总导入到IoC容器并配置好。有了自动配置类，免去了手动编写配置、注入功能组件等工作。

    给容器中导入组件，组件为导入组件的选择器

  • @ComponentScan 扫描，定义扫描的包路径

    标注了需要自动扫描并加载符合条件的组件或 bean 定义，最终将这些 bean 定义加载到容器中。加载 bean 定义到 Spring 的 IoC 容器，我们可以手工单个注册，不一定非要通过批量的自动扫描完成，所以说@ComponentScan是可有可无的。

Error

不能找到SpringBootApplication注解中的proxyBeanMethods类，或者是不能找到LogMessage类

原因是导入的jar包版本不对，如2.1.3包含这些类但2.3.2中不包含

SpringBoot的SpringBootApplication的jar包放在maven仓库中org\springframework\boot\spring-boot-autoconfigure\2.1.3.RELEASE\spring-boot-autoconfigure-2.1.3.RELEASE.jar；同理SpringBoot放在E:\maven-reposity\org\springframework\boot\spring-boot\中

不过非常诡异的是，在IDEA中导包的时候，尽管选的是2.1.3版本，但是导进去的一直是2.3.2版本

原因好像是2.1.3版本虽然在项目中，但是不在admin的module中，需要在项目结构中设置add to module将jar包添加到module中
