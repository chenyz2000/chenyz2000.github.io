HTTPS

Posted on 2023-09-18 | Post modified: 2023-09-18 | In 其他

HTTPS = HTTP + SSL/TLS，使用HTTP通信，使用SSL/TLS加密数据包，SSL/TLS在HTTP层和TCP层之间

SSL是TLS的前身，SSL3.0和TLS1.0存在漏洞且已经废弃，目前使用广泛的是TLS1.1和TLS1.2

不加密的风险：1、窃取信息，2、修改信息，3、冒充身份参与通信

对称加密：加密和解密用的都是同一个密钥

非对称加密：加密和解密使用不同的密钥

非对称加密性能较低，但安全性很强

加密方法：先非对称加密，后对称加密

  1. 服务器拥有用于非对称的公钥和私钥，公钥由证书保证
  2. 客户端向服务器索要公钥，客户端随机生成一个用于对称加密的密钥，然后用公钥加密后传给服务器
  3. 服务器收到密文后，使用服务器端的私钥解密，得到对称加密的密钥

如何防止公钥被篡改？

将公钥放在数字证书中，只要证书是可信的，公钥就是可信的

思考题：

  1. 为什么要用对称加密+非对称加密？
  2. 为什么不能只用非对称加密？
  3. 为什么需要数字证书？
  4. 为什么需要数字签名？

关于数字证书、数字签名的细节见https://zhuanlan.zhihu.com/p/43789231
