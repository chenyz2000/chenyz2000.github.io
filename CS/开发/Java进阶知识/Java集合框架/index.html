Java集合框架

Posted on 2020-07-19 | Post modified: 2020-07-19 | In Java进阶知识

简介

Java2之前，Java工具包提供了一些简单的可以自扩展的容器类，包括Enumeration，BitSet，Dictionary，Vector，Stack，Hashtable和 Properties

在Java2中，设计了一个核心统一的主题，引入了一套完整的集合框架

旧的容器类目前仍保留，但应尽量使用新的集合框架

体系

分为三层

  • 接口：抽象数据类型
  • 实现类（抽象类+具体类）
  • 算法

面试常考知识点

Collection、Map； List、Set、Queue

ArrayList和LinkedList

ArrayList和Vector，扩容机制

HashMap和HashTable，底层原理，红黑树的算法

线程安全和不安全的集合类

ConcurrentHashMap

HashMap底层原理

1.7及以前采用数组+链表，1.8及以后采用数组+链表+红黑树

数组是一个Entry<K,V>数组或者Node<K,V>数组，数组名叫Table

1.7和1.8的区别

  • 1.7只用链表，1.8除了链表还用红黑树，最坏情况的复杂度从O(n)降为O(logn)

  • 1.7的K-V对被封装为Entry对象，1.8的K-V对被封装为Node对象

    Entry是接口，Node是其实现类

  • 1.7采用头插法，1.8采用尾插法，因为1.8插入时需要遍历一遍链表统计元素个数

存放过程

  1. 对Key进行hash计算，得到hash值

    这里得到的hash值为(h=key.hashcode()) ^ (h>>>16)，即hashcode与自己右移4位进行异或。(>>>16相当于>>4)
    这样比直接用hashcode进入下一步更好，因为下一步取模会丢失hashcode的高位信息；像这样高位低位异或，能保留高位信息，减少碰撞。

  2. 对hash值进行取模（这里采用了一个小算法：和length-1进行位与运算，length一直为2的n次方），得到Key的下标

  3. 将封装的K-V对象（Entry/Node）放在对应位置

    • 如果对应位置为空，直接存放

    • 如果不为空，JDK1.7→→先判断是否需要扩容，然后头插

    • 如果不为空，JDK1.8，新节点为红黑树Node→→加入红黑树，如果已存在Key则更新Value，加入后判断是否需要扩容

    • 如果不为空，JDK1.8，新节点为链表Node→→尾插加入链表，如果已存在Key则更新Value，加入后判断是否需要扩容或转为红黑树

      先插入节点，然后若数量大于8（即第9个结点），则链表转红黑树

扩容

扩容指的是数组扩容

默认值

数组默认大小为16：static final int DEFAULT_INITIAL_CAPACITY = 1 << 4;

负载因子（数组扩容的临界比例）为0.75：static final float DEFAULT_LOAD_FACTOR = 0.75f;和泊松分布有关

链表转红黑树的临界值为8：static final int TREEIFY_THRESHOLD = 8;也和泊松分布相关。在理想情况下，链表长度符合泊松分布，各个长度的命中概率依次递减，当长度为8的时候，概率仅为0.00000006，小于千万分之一。所以通常情况下，不会发生从链表向红黑树的转换。除非极端情况，比如人为重写hashcode方法，让节点易碰撞。

红黑树转链表的临界值为6：static final int UNTREEIFY_THRESHOLD = 6;
