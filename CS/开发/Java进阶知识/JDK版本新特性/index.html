JDK8——LTS版本，2014年

函数式接口

只包含一个抽象方法的接口被称为函数式接口

JDK8之前常用的函数式接口有Runnable、Callable、Comparator

JDK8在java.util.function包下添加了的丰富的函数式接口，包括：

  • Function：一个参数，一个返回值
  • Supplier：无参数，一个返回值
  • Comsumer：一个参数，无返回
  • Predicate：一个参数，一个布尔返回值
  • Operator

Lambda表达式

Lambda表达式的本质是作为函数式接口的实例

形式参数->方法体，值为一个函数式接口对象

创建Runnable接口的举例

方法引用

双冒号（::）运算符在JDK8中被称作方法引用method reference

方法引用实质上是在特定情况下对Lambda表达式的缩写形式，也是函数式接口的实例

类型 方法引用语法 对应的Lambda表达式 举例
静态方法 类名::静态方法名 (args) -> 类名.静态方法名(args) System::currentTimeMillis
实例方法 实例名::实例方法名 (args) -> 实例名.实例方法名(args)
特定类的任意对象方法 类名::实例方法名 (实例名, args) -> 类名.实例方法名**(args)** Arrays.sort(arr, String::compareTo);
类构造器、数组构造器 类名::new
类型名[]::new		 (args) -> new 类名(args)
(args) -> new 类型名[args]		 String[]::new

注意：特定类的任意对象方法引用，要求调用处函数式接口的方法比引用的方法多一个参数，且该参数位于首位、是调用引用方法的对象，该参数将作为方法的调用者

方法引用一般配合函数式接口使用，如果某个方法参数和函数式接口对应，就可以直接传入方法引用给函数式接口
