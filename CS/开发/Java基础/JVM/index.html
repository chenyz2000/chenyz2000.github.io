JVM

Posted on 2021-10-04 | Post modified: 2021-10-07 | In Java基础

JVM是C语言写的

JVM体系结构

栈、本地方法栈、程序计数器——这三块不会有垃圾需要回收

所谓的JVM调优，就是在方法区和堆中调（方法区也是特殊的堆）

本地方法，用native关键词

类加载器

作用：解析Class文件，生成class对象

类的加载过程

  1. 加载Load：将.class字节码文件读入内存，静态数据转换为方法区运行时的数据结构，生成Class对象到堆中，作为方法区中类数据的访问入口，此过程由类加载器完成
  2. 链接Link：将类的二进制数据合并到JRE（JVM的运行环境）中。
    1. 验证：首先检查类信息是否符合JVM规范
    2. 准备：之后在方法区中为类的static变量（非final常量）设置内存和默认值（0或null等，初始化时才赋指定值），为类的final常量设置内存并赋指定值、存放到常量区中
    3. 解析：最后将类的符号引用（变量名、方法名）替换为直接引用（内存地址）
  3. 初始化Initialize：JVM对类进行初始化。JVM会执行<cInit>()类构造器方法对类进行构造（与构造函数创建对象不同），其中<cInit>()由编译器收集static代码（包括静态的变量、方法和代码块）合并产生。JVM会保证<cInit>()方法在多线程环境中被正确加锁和同步

类加载器ClassLoader

类加载器的类型：

  • BootStrapClassLoader：引导类加载器（根加载器），负责Java核心类库rt.jar的加载，由C++编写，是JVM自带的类加载器，开发者无法直接获取此加载器
  • ExtClassLoader：扩展类加载器，负责jre/lib/ext扩展目录下类的加载
  • AppClassLoader：当前应用程序加载器（系统类加载器），最常用的加载器，负责java -classpath目录下的类的加载
  • 自定义类加载器：实现ClassLoader抽象类

四者由上到下是父类到子类的继承关系，检查类是否已装载的时候由下到上进行，加载类的时候由上到下进行

类缓存：类加载的时候会在缓存中维持一段时间，之后类再加载的时候会向缓存中查找是否有之前加载的类。JVM垃圾回收机制gc会自动回收这些缓存

双亲委派机制：收到类加载请求时，从下到上进行检查，若已装载则结束；若未装载，则把请求委托给父类加载器进行检查和装载。目的是防止系统级别的类被篡改替换。

沙箱安全机制：Java安全模型的核心就是沙箱，把Java程序限制在特定的运行范围中，限制程序对本地系统资源的访问。Java中执行程序分为本地代码和远程代码，本地代码默认可信任，远程代码默认不可信任

通过反射，class对象获取类的加载器

类的初始化

主动引用类的时候会发生初始化，包括

  • new一个类
  • 虚拟机启动时初始化main方法所在的类
  • 调用类的静态变量（除了final常量）和静态方法（只有声明这个静态域的类会初始化，如通过子类调用父类声明的静态域，子类不会初始化）
  • 用java.lang.reflect包的方法对类进行反射调用（用类.class属性不会导致类被初始化）
  • 初始化类时，若其父类没有初始化，则先初始化父类

被动引用类时不会发生初始化，包括

  • 创建类的数组
  • 调用类的final常量时类不会初始化，因为常量在链接阶段就在常量池中了

运行时数据区

栈stack——线程私有

存放基础数据类型，如Boolean、Byte、Char等

以及对堆中对象的引用

堆heap——线程公有

存放new出来的对象和数组，如String

也存放JVM自动创建的Class对象

方法区method area——线程公有

是特殊的堆

包括虚拟机加载的类，和运行时常量池

类的信息包括信息（类名、方法信息、字段信息、接口信息等）、类的常量池表（数值、字符串值、类引用、字段引用、方法引用）、类变量、类方法、指向Class对象的引用、指向类加载器的引用

类加载后，类常量池表将载入到方法区的运行时常量池中

元空间metaspace和永久代premGen

方法区是JVM抽象的规范，元空间和永久代是方法区的具体实现。1.8之前的实现方式是永久代，当时永久代是堆空间的一部分；1.8及以后从堆空间里独立出来了元空间，使用了本地内存（系统的内存）而不是堆内存。

程序计数器program counter register——线程私有

记录当前需要执行的指令地址，即每个线程执行到哪了

空间占用很小，以至于没有定义的异常

本地方法栈native method stacks——线程私有

给本地方法提供的专门的栈空间

本地方法指其他语言编写的、交由Java运行的方法

用native关键字描述的就是本地方法，如thread.sleep()
