Java多线程

Posted on 2021-11-25 | Post modified: 2021-11-25 | In Java基础

线程创建

方法一 继承Thread类

步骤：继承Thread类→重写run()方法→创建线程对象→调用start()方法启动线程

线程只能启动一次，启动后处于就绪状态等待CPU调度

实质上，Thread类本身实现了Runnable接口

方法二 实现Runnable接口（最推荐）

因为Java的单继承规则，所以更推荐实现Runnable接口

步骤：实现Runnable接口→实现run()方法→创建实现类的对象obj→把obj传入构造函数创建Thread对象→调用start()方法启动线程

使用lambda表达式化简

测试中常用到：线程名称

方法三 实现Callable接口（需要用到线程池）

步骤：实现Callable接口→实现call()方法→创建实现类的对象→四个步骤创建服务、提交执行、关闭服务

另一种启动

方法三的好处是可以获取结果、可以抛出异常

操作线程

线程状态

  • 创建
  • 就绪：创建的线程对象调用start()进入就绪
  • 运行
  • 阻塞：运行时调用wait、sleep、同步锁定，进入阻塞
  • 结束

t.getState()获得常量，属于枚举类型Thread.State，表示虚拟机线程的状态，不是操作系统线程的状态

  • NEW
  • RUNNABLE
  • BLOCKED：阻塞等待监视器锁定
  • WAITING：阻塞等待其他线程执行特定动作
  • TIMED_WAITING：阻塞等待其他线程到指定时间
  • TERMINATED

线程控制

  • 线程插队：t.join()，当前线程阻塞，t线程插队，其他线程不受影响，t线程执行完再执行当前线程

  • 线程让步：t.yield()，暂停t线程，运行转为就绪状态，让CPU重新调度（能否成功让步还是要看CPU）

  • 停止线程：JDK提供的停止方法已经废弃，推荐设置标志位flag，想要停止时设置flag为false，线程主动停止

  • 线程休眠：对当前线程使用Thread.sleep()，sleep不会释放对象的锁

t.isAlive()判断线程是否存活

优先级

t.setPriority(num)，值为1~10，1最低10最高，默认优先级为5

t.getPriority()

优先级高只是权重大，少数情况下CPU不会按优先级顺序执行线程

建议先设置优先级再启动线程

守护线程

t.setDaemon(true)

线程分为用户进程和守护进程，虚拟机确保用户进程执行完毕，不等待守护线程执行完毕

守护线程如：后台操作日志、监控内存、垃圾清理gc

线程同步（重点）

多个线程对同个资源进行操作，数据会紊乱，需要并发控制

线程同步就是等待、排队

synchronized，锁机制，实现原理是锁+等待队列

加锁保证了安全性，但会影响性能

同步方法

锁定对象是调用该同步方法的对象（this）

加了synchronized的方法一定要获得锁定对象的锁才能执行

同步块

锁定对象是同步监视器obj

加了synchronized的方法一定要获得锁定对象的锁才能执行

当锁定对象不是this时，只能用同步块的方法

显式锁Lock

ReentrantLock可重入锁实现了Lock接口

Lock是显式锁，需要加锁解锁；synchronized是隐式锁，出了作用域自动解开

Lock只能锁代码块；synchronized可以锁代码块也可以锁方法

Lock的效率更高

建议使用优先顺序：先Lock，再同步代码块，再同步方法

JUC

java.util.concurrent并发包

CopyOnWriteArrayList线程安全的List类

volatile：关键字，变量对所有线程都可见

transient

线程通信

Object类的方法，只能在synchronized中使用

obj.wait()等待，直到其他线程通知，会释放锁

obj.notify()唤醒

缓冲区

线程池

经常创建、销毁线程对性能影响非常大

线程池当中存储了很多线程，用时直接获取，用后放回池中

Executors：线程池的工厂类，用来返回不同的线程池

ExecutorService：线程池接口，常用子类ThreadPoolExecutor，具有以下方法

  • void excute(Runnable command)
  • Future submit(Callable task)
  • void shutdown()
