Nodejs

Posted on 2021-01-28 | Post modified: 2021-01-28 | In Nodejs

简介

Node.js是一个能在服务器端运行JS代码的开源的、跨平台的JS运行环境

从浏览器到服务器，原本只可以对浏览器进行操作，现在可以用JS独立写程序对系统进行操作，让JS和Java、C等编程语言地位一样了

JS标准定义的API只为构建浏览器的应用程序，而commonJS制定了一个更广泛的标准库，类似Java、Python的规范，使JS不仅仅停留在浏览器脚本的阶段

CommonJS是模块化的标准，Nodejs是CommonJS的实现。

采用谷歌V8引擎，使用事件驱动、非阻塞和异步I/O模型等技术，即客户端请求不等待数据库IO

Nodejs是单线程，所有进来的请求共用同一个线程，擅长高并发

单线程有优点有缺点，缺点是不能处理访问量过大，除非分布式布置多台服务器提供多个线程，目前大访问量还是以Java为主

Nodejs是对ES标准的一个实现，可以使用基本数据类型、数组、对象、函数，但是不能使用DOM

Ryan Dahl在2009年开始开发，原名叫web.js，目的是为了开发一个高性能的web服务器

后来Nodejs生态越来越庞大，现在它作为web服务器的功能稍微弱化了，而是用来使用该生态中的一些插件、软件、框架（如webpack就是使用了Nodejs）

Nodejs版本原先是零点几，之后产生了一个分支io.js，之后两者合并，版本变为4.0，此后稳定版都是偶数，6.0,8.0

热部署工具

supervisor或nodemon

npm install -g supervisor

之后运行命令时用supervisor代替node运行js文件

当程序更新或结束时，supervisor会将它重启（所以如果程序不是保持运行，就会不断刷新）

自定义模块

CommonJS中规定，可以把公共的功能抽离成为一个单独的JS文件作为一个模块。

默认模块里的属性和方法外面是无法访问的

如果想要外界可以访问模块里的属性方法，就要用exports或module.exports使之暴露出来

在需要使用的其他模块中，使用require的方法导入模块
