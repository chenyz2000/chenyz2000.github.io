Linux基本命令

Posted on 2020-02-22 | Post modified: 2020-07-29 | In Linux

命令大全

命令格式

格式：命令 -选项 参数

多个选项时，可以写在一起

-调用简化选项，–调用完整选项

清屏命令ctrl+L或clear

终止命令ctrl+C

目录处理

ls

list

  • -a（–all）显示全部文件，包括.开头的隐藏文件

  • -l显示文件具体信息

    信息分别为文件类型和权限，文件计数，所有者，所属组，字节数，修改时间（linux中没有创建时间这个概念），文件名

    其中文件类型和权限：

    第一个字母表示文件类型，-文件，d目录，l软链接。

    后九位分三组，表示所有者u、所属组g、其他人o的权限。每一组会有rwx三个字母，r读，w写，x执行，没有某个权限则为-

    目录的rwx表示可以列出目录中的内容、可以在目录中创建删除文件、可以进入目录；文件的w表示可以修改文件（非创建删除）

    文件的默认所有者为创建用户，所属组为该用户的缺省组

  • -h人性化显示文件大小

  • -d查看目录而不是目录内部

  • -i查看文件i结点

mkdir

make directory

可以相对路径可以绝对路径，只要路径最后一层是要新建的目录

  • 只能创建一层，想递归创建需要-p

cd

change directory，是shell内置命令

基本上命令都在/bin中，除了cd

.表示当前目录 …表示上一级（linux里面cd要空格，dos不需要）

pwd

print working directory

rmdir

remove empty directory

只能删除空目录

cp

copy拷贝文件，复制和改名

cp [-rp] 源文件或目录 目标目录

原文件或目录可以是多个，目标目录只有一个

目标目录不存在的话，复制并改名（相当于创建目标目录）

  • -r复制目录
  • 拷贝过去的文件，修改时间会改变，-p保留文件属性防止变化

mv

move，剪切和改名，类似cp一样

只改名就等于在同一个文件中移动并改名

rm

remove删除文件

  • -r（–recursive）可对目录操作
  • -f（–force）强制执行，不询问是否删除
  • -i询问确认

文件处理

touch

若文件不存在则创建空文件，若文件存在则修改文件访问时间

和mkdir类似，可以相对路径或绝对路径创建，只要路径最后一层是要新建的文件

可以创建多个文件

linux文件名和windows相比，可以包含一些标点，但不建议包含空格（含空格的文件名需要用双引号括住）

cat

浏览文件内容

  • -n显示行号

tac

倒着浏览文件内容，没有-n选项

more

分页显示文件内容，空格键或f翻页，enter键换行，q键退出

less

包含more的功能，且可向上翻页，可搜索

上箭头一行一行，pageup一页一页

显示文件内容时，/加搜索内容可向下查找，？可向上查找，n查看上/下一个

  • [ ] less不能读二进制文件？

head

只看文件前几行

示例head -n 20 /etc/services

  • -n指定行数，不加-n默认十行

tail

查看文件末尾几行，类似head

  • -f动态显示，不回到命令行，多用于监控日志

链接处理

软链接文件权限不可改，对全部用户都可读可写可运行，和源文件权限无关。软链接相当于win中的快捷方式，属性中能看到它指向的源文件。软链接也可以有它的软链接。

硬链接像是cp -p，但是两个文件可以同步更新。文件id值和源文件一样（同块内存映射多个文件）。

软链接和硬链接的区别：源文件删除后，软链接提示找不到文件，硬链接依然存在；硬链接不能跨分区，软链接可以；硬链接不可以针对目录，软链接可以。

ln

link

ln -s 源文件 新文件

  • 含-s为soft软链接，不含则为hard硬链接

权限管理

文件权限只可由文件所有者和管理员更改

chmod

change mode

chmod [ugoa +-= rwx，逗号分隔] 文件或目录

chmod 3个数字 文件或目录，其中每个数字是每组rwx对应的二进制的十进制值，如rwxrw-r–对应764

  • -R递归修改，要大写

修改软链接权限时，修改的是源文件

chown

change owner更改所有者，/bin/chown，但只有root可以操作

chown 用户 文件

chgrp

change group更改所属组，和chown类似

umask

user file-creation mask，创建文件时的缺省权限掩码（ugo没有哪些权限），shell内置命令

umask [-S] [权限掩码]

ugo默认都没有x权限

  • -S以rwx形式显示权限掩码，不加S是显示1位特殊权限（不管）+3位分别表示ugo权限（每一位值是rwx对应的二进制的十进制值）的掩码

目录的权限与umask显示一致？？啥意思

文件搜索命令

find

find 目录 [选项 匹配条件] [-a/-o 选项 匹配条件]

  • -name根据文件名搜索（精准搜索），-iname不区分大小写

    *匹配任意个字符，？匹配单个字符

  • -size文件大小，条件为多少数据块（0.5kb），数字前可加±表示大于小于

  • -type根据文件类型查找，匹配条件f表示文件，d表示目录，l表示软链接文件

  • -amin访问时间，-cmin文件属性改变时间，-mmin文件内容改变时间，匹配条件为多少分钟，数字前可加±表示超过多久之前、多久之内

  • -user根据所有者，-group根据所属组

  • -inum根据文件i结点

-a表示and，-o表示or

后面还可跟-exec 命令 {} \;或-ok 命令 {} \;对找到的结果进行某个命令的操作，区别是-ok有询问-exec没有，命令可以是如显示详细信息或删除

locate

/usr/bin/locate

  • -i不区分大小写

系统建立文件资料库/var/lib/mlocate/mlocate.db，在其中进行搜索，对系统资源消耗较少，类似win中的everything

可以通过updatedb更新资料库

资料库只收录部分内容，如/tmp不在收录范围内

which

/usr/bin/which

搜索命令所在目录和alias别名

whereis

/usr/bin/which

搜索命令或配置文件所在目录和帮助文档路径

grep

显示文件内字符串所在的行数

grep -iv 字符串 文件

  • -i不区分大小写
  • -v排除指定字符串所在行

帮助命令

man

manual，/usr/bin/man

man 命令或配置文件（名称不是路径）

浏览模式和more一样，还带有less的搜索功能

当有命令与配置文件重名时，优先查看命令的帮助文档（1<5）

如果想看配置文件的帮助，可以man 5 passwd

whatis

查询命令的简短信息，即名字的含义，后面跟命令的名称

apropos

查询配置文件的简短信息，类似whatis

–help选项

列出命令的选项

help

查看shell内置命令的帮助信息

用户管理命令

useradd

/usr/sbin/useradd，只有管理员可以操作

passwd

管理员可以设置简单密码，普通用户不可以

管理员可以改任何人的密码，普通用户只可以改自己的密码

who

查看当前用户

tty表示本地登录，pts表示远程登录

w

第一行显示信息和命令uptime结果一致：当前时间，系统运行时间，用户数量，系统1分钟、5分钟、15分钟平均负载

下面的用户信息除了who的结果外，还有idle用户无操作时间，JCPU累计占用cpu时间，PCPU当前命令占用cpu时间，what当前操作

压缩解压命令

Linux的压缩格式win基本支持，但rar为linux不支持，zip为二者都支持

gzip

GNU zip

后面直接加文件名，产生压缩文件，不保留源文件

不能处理目录

gunzip

GNU unzip

解压缩，和gzip对用

等同于gzip -d

bzip2

/usr/bin/bzip2

gzip的升级版本，压缩比胜过gzip，且多了-k选项可以保留源文件

生成.bz2文件

bunzip2

同理，且有-k选项保留压缩文件，同理等于bzip2 -d

tar

压缩tar [-cvzjf] [压缩后文件名] [目录]

压缩后文件只能为一个

  • -c打包
  • -v显示详细信息
  • -z打包同时压缩，直接生成压缩后的打包文件，相当于tar加gzip
  • -j以bzip2的方式压缩
  • -f指定文件名，此选项只能写在最后面

解压tar [-xvzjf] [压缩文件名]

  • -c对应变成-x，其他选项一样

z和j压缩解压的格式需要对应，不能混用

zip

/usr/bin/zip

zip [-r] [压缩后文件名] [文件或目录]

  • -r压缩目录

会保留源文件

unzip

/usr/bin/unzip

unzip [文件名]

网络命令

write

/usr/bin/write

write 用户名

给登录服务器的用户发消息，ctrl+d保存并结束输入状态

wall

write all，/usr/bin/wall

wall 消息

给在线的所有用户发送广播消息，包括自己

mail

同write，但可以发送给不在线的用户

只写mail可以查看自己收到的邮件

ping

ping [-c num] ip地址

  • -c指定发送次数

ifconfig

interface configure，/sbin/ifconfig

ifconfig 网卡名称 ip地址

查看和设置网卡信息

网卡信息：link encap网络类型，一般为以太网；HWadder，网卡硬件地址，即Mac地址；iner addr当前ip地址；bcast当前网络地址；Mask子网掩码；inet6 addr是ipv6的地址信息；RX packets数据包接收量，TX packets发送量；RX bytes数据包接收大小，TX packets发送大小；base address网卡在内存中的物理地址

last

显示计算机用户的所有登录信息，也能显示重启记录

lastlog

每个用户最后登录的信息

  • -u加用户uid只查看某个用户

traceroute

显示数据包到主机间的路径

netstat

查询网络状态

  • -t，-u分别表示TCP协议，UDP协议

    传输协议：tcp面向连接更稳定安全（http属于tcp），udp不监听更快速

  • -l查看监听，-r查看路由

  • -n以IP地址和端口号显示，而不是以域名显示，减少了解析时间

  • -a所有

常见用法有：

-tlun查看本机已开启（正在监听）的服务端口，主要目的是看端口号

-an查看本机所有的网络连接（正在监听的服务+已连接的服务+系统程序）

-rn查看本机路由，结果的gateway中显示了网关

setup

/usr/bin/setup，需要root权限

配置网络，永久生效，是Redhat开发的管理工具集合

DHCP的方框中打星号，相当于自动获取ip，但是需要当前网络环境有DHCPserver服务

配置后需要service network restart

挂载命令

mount

挂载命令，linux可以自动挂载硬盘，但移动设备需要手动挂载

mount [-t 文件系统] 设备文件名 挂载点

光盘的文件系统是iso9660，不写也行

设备文件名默认/dev/sr0

创建一个空目录作为挂载点，一般在/mnt下

umount

umount 设备文件名或挂载点（写一个就行）

当前路径在挂载点内部时不能使用命令

关机重启命令

shutdown

shutdown 选项 时间

  • -h表示关机
  • -r表示重启
  • -c表示取消上个命令

时间可以为几点几分

shutdown是最早的可以安全关机的命令，建议使用

其他关机命令

halt

poweroff

init 0

系统运行级别

0~6分别为：关机，单用户（安全模式只启动核心程序用于做修复），不完全多用户不含NFS服务（系统间文件共享），完全多用户，未分配，图形界面，重启

配置文件为/etc/inittab，但centos7以上不支持在配置文件修改了

命令runlevel，显示之前级别和当前级别

其他重启命令

reboot

init 6

logout

logout

执行历史命令

  • !!：重复上一条命令，相当于!-1
  • !-n：重新执行倒数第n条历史命令
  • !n：重新执行第n条历史命令
  • !string：重新执行以字符串string开头的最近的历史命令
  • !?string? ：重新执行最近的、包含字符串string的那条历史命令
  • !# ：到现在为止所输入的整个命令行
