Linux图形界面

Posted on 2021-03-16 | Post modified: 2021-06-09 | In Linux

:heavy_check_mark:图形化接口协议

  • SSH+X11转发（首选）
    Linux系统启用X协议，配置X Client，本地有桌面环境的机器（如Windows）配置X Server
  • RFB：Remote Frame Buffer，VNC常用的协议（推荐）
    需要Linux系统安装X Windows图形界面
  • Xdmcp：透过X display manager（xdm, gdm, kdm等）所提供的功能协议
  • RDP：Remote Desktop Protocol，windows系统的默认协议
  • 其他：FreeNX，Xpra，向日葵，teamviwer

xmanager支持XDcmp和XStart（SSH+X11）两类会话

vnc是远程连入操作系统，所有操作在UNIX、Linux主机服务端进行，即使操作中网络断开，也不影响操作的进行

xmanager通过端口将主机服务器的UI界面引导到本地电脑进行展现，如果操作中网络断开，操作将失败

vnc在远程端上占用更小，xmanager更清晰

SSH+X11转发

服务器端不用安装服务和进行设置，开启SSH即可，只要求服务器有桌面环境如Gnome。

:heavy_check_mark: 服务器端SSH配置

配置/etc/ssh/sshd_config

:heavy_check_mark:Windows客户端方案1——xshell+VcXsrv（较流畅，画质良好，启动7s，延迟3s内，未解决的问题：图像占了全屏）

xshell可以用putty代替， VcXsrv可以用Xming或Xmanager代替

VcXsrv和Xming是windows上的X Server，是开源软件；Xmanager是商业软件

xshell配置—连接—SSH—隧道—X11转移，选中转发X11连接到，X Display（用Xmanager就选Xmanager），填localhost:0.0

远程命令行连接上之后，敲一个程序的命令（如下），图形化界面定向到windows上了

  • xterm，gedit，xclock

  • xfce4，gnome-session，startkde

    gnome-session只能运行一个，如果要在远程运行，服务器就只能打开在命令行界面

:heavy_check_mark:Windows客户端方案2——Xstart（卡顿，画质高，启动10s，延迟5s）

Xstart在Xmanager中有

类似上一个，不过没有用xshell，而是直接用Xstart

Xstart是一个窗口化的登陆界面，支持很多协议，如常用的SSH、Telnet等

在Xstart中设置好服务器的ip、用户后，设置命令，此处的命令为上一个方案中的命令

客户端为Linux的配置（理论上）

  1. 配置/etc/ssh/sshd_config
  1. 运行命令xhost + 172.16.1.45，允许远程X界面连接过来：(注意+左右两边的空格)
  2. 用ssh -X username@172.16.1.45进行图形化连接
  3. 运行export DISPLAY=localhost:0.0，也有可能是10.0，表示将远程X的显示界面定位到本地的0.0显示器
  4. 验证，xclock或gedit等，能够显示窗口

以下是成功连接Ubuntu的例子，非常流畅

Gnome3和Unity桌面，由于兼容性问题，VNC连接后会什么也无法显示，只能使用Gnome2或xface

apt install xfce4

Ubuntu服务端

  1. 安装VNC Server，apt install vnc4server

  2. 配置VNC，vim ~/.vnc/xstartup

    或者好像简短的写法

    好像使用tigervncserver，要改的是/etc/X11/Xvnc-session文件

  3. 添加权限，chmod +x ~/.vnc/xstartup

  4. 运行VNC， vncserver，第一次运行需要输入密码

  5. 结束服务vncserver -kill :1

客户端

安装RealVNC、VNCViewer等

客户端连接ip:5901

vncserver自启动！！

因为安装vncserver，它自己不带.service服务文件

所以要自己在/lib/systemd/system/下创建vnc.service文件

之后可以通过service命令运行，通过systemctl命令自启动

ps：这个服务不是一直运行的，只是运行一下就产生一个server，然后service结束

X Window和VNC讲解参考

X Window原理

https://blog.csdn.net/a379039233/article/details/80782351

Linux图形界面基于X Window System（X，X Window）（先不管wayland）。X Window System采用C/S架构，分为X server和X client两部分，这两部分通过X协议（X11）进行交互，可以基于网络通信，也就是说它们可以不在同一台机器上。所以X天生支持远程图形界面，不过代价是在本地跑的时候由于通信性能会有一定损失

X server负责管理驱动机器的显卡等硬件，并在屏幕上绘制和显示（控制分辨率、鼠标样式等），捕捉鼠标键盘等操作。X Server包括底层的和设备相关的图形库，目前主要是Xlib。X Server的具体实现有Xorg、XFree86等

X client负责消息处理和计算，不关心具体如何绘画，接收server的操作，告知server要绘制的内容。任何一个X应用程序，包括Windows Manager都是X client

Windows Manager（WM）：一个client端应用程序，主要功能是移动窗口、改变窗口大小、最小化窗口等。如KWin是KDE的WM

Gnome、KDE不是WM，也不是X Window，它们是桌面环境或桌面系统，包括应用程序和函数库

工作流程

  1. 用户使用鼠标键盘对server下达指令
  2. server将消息传递给client
  3. client进行程序运算
  4. client将结果传递给server
  5. server将结果显示在屏幕上

VNC原理

远程访问图像UI，可用于X11、Windows、Mac

只允许键击等消息通过网络传输，对客户端要求小

Linux四大图形界面

Gnome、KDE、Xfce、LXDE（频率由多到少）

GTK和Qt都是Widget集，用于构造按钮、滚动条、菜单等控件

KDE使用Qt库开发，Gnome使用GTK+库开发，XFCE基于GTK，LXDE基于GTK

Cinnamon基于GTK，MATE基于GTK，Trinity基于Qt

Qt库和GTK库是基于X window server的，需要遵守x11协议，在x window server上运行，作为client应用实现的基础类库

XDCMP

XDMCP（X Display Manager Control Protocol）X 显示监控协议，也是Xmanager使用的协议之一。
通常使用的是UDP端口：177

用XDMCP连接，服务器端得做一些配置，麻烦

配置gnome，/etc/gdm3/custom.conf

yum安装XWindowyum groupinstall "X Window System"

xwindow和xdmcp区别？

ubuntu18.0后使用gdm3(gnome display manager)桌面，这个桌面有硬件优化不能使用xdcmp，想用xdcmp需要把gdm3桌面换成lightdm桌面
