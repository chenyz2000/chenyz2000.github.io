版本控制的作用

  • 管理版本，随时回退到任何时刻
  • 协同开发，自动合并代码

文件状态

四个状态：

  • 初始状态是untracked
  • add后成staged
  • commit后成unmodified
  • 有提交记录的文件修改后成modified

git status查看当前仓库的状态信息，红色是未暂存，绿色是已暂存

暂存的文件修改后，状态会变成红色

git diff能查看已写入暂存和有改动却未写入暂存的区别

git reset可以取消暂存

commit只提交已暂存的文件

log查看历史

git log

–oneline参数显示简洁版本，–graph显示分支与合并，–all显示所有分支

显示每个commit的哈希值、提交者、时间、信息

哈希值很长，但是只要前七位就能锁定

reset回退和revert反做

reset

git reset <Node>将当前分支状态改到指定的Node

该版本之后的版本不保留在当前版本控制中，不过之后的版本也能通过git reflog查到所有commit的七位commitID

reset默认采用–mixed方式，保留文件变更，但不保留提交，变更内容处于modified状态；参数–hard不保留文件变更；–soft保留文件变更且变更内容处于staged状态

revert

git revert <Node>

在当前分支后新建一个子结点，该结点状态与指定的Node之前的状态相同（不修改历史提交，可以与他人同步）
