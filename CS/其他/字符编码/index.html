字符编码

Posted on 2020-05-29 | Post modified: 2020-05-29 | In 其他

计算机中的字符编码

步骤

  1. 确定包含哪些字符
  2. 进行数学编号
  3. 映射为二进制，确定逻辑编号
  4. 通过编码方案，确定物理存储编码

编码字符集和编码方案

  • 编码字符集：字库表中所有字符和其对应编码的集合（这个编码和计算机无关）

  • 编码方案：（最简单的形式就是直接用字符集编码后的二进制值）

Unicode字符集：0000-FFFF为0号平面，10000-1FFFF为1号平面，…，A0000-AFFFF为10号平面，…，F0000-FFFFF为15号平面，100000-10FFFF为16号平面

UTF-8编码规则：1字节 0xxxxxxx，2字节 110xxxxx 10xxxxxx，3字节 1110xxxx 10xxxxxx 10xxxxxx，4字节 11110xxx 10xxxxxx 10xxxxxx 10xxxxxx，5字节 111110xx 10xxxxxx 10xxxxxx 10xxxxxx 10xxxxxx，6字节 1111110x 10xxxxxx 10xxxxxx 10xxxxxx 10xxxxxx 10xxxxxx

"严"字的Unicode码是100111000100101，UTF-8编码是11100100 10111000 10100101

编码字符集

ASCII编码字符集——ASCII编码方案

范围0~127，最高位统一为0

扩展编码字符集

加字节，是ASCII的超集

  • 西欧语言：ISO 8859(1-15)，范围0~255

  • 中文：GB2312+Big5 ⊆ GBK ⊆ GB18030

    其中GB2312范围0xA1A1~0xFEFE，双字节表示——GB2312的编码方案为EUC-CN

    Big5为繁体中文

  • 日文：Shift_JIS

Unicode编码字符集

三种编码方案

  • UTF-8：兼容ASCII，1-4个字节，变长
  • UTF-16：2或4个字节，变长
  • UTF-32：4个字节，定长

ANSI：windows上非Unicode的默认编码，记事本默认采用ANSI存储。简体中文系统中代表GBK，繁体中文系统中代表Big5，日文系统中代表Shift_JIS

发展历史：美国人发明ascii，用七位表示英文字符；后来欧洲人发现法语、希伯来语、俄语符号没法表示，所以把ascii码最高一位的1编进来，形成了15套编法，这15套中的128~255都不一样；之后中国人需要表示汉语，所以发明了GB2312表示常用的汉字（65536），有些生僻字不能显示，之后又对GB2312进行了升级；后来，人们发现文件需要全球使用，两边字符集不同，所以发明了万国码Unicode；不过Unicode统一了全世界字符的二进制编码，但没有规定如何存储，所以出现了Unicode的多种存储方式

中英文字符所占字节数

英文

GB2312、GBK、GB18030：1字节

ISO-8859-1：1字节

UTF-8：1字节

UTF-16：4字节

中文

GB2312、GBK、GB18030：2字节

UTF-8：3字节

UTF-16：4字节

Java中的编码

Java源文件建议用UTF-8，文件操作建议UTF-8

Java程序内部使用UTF-16（不由程序员控制）

修改CMD编码

暂时

查看编码的命令chcp
修改为UTF-8chcp 65001
修改为GBKchcp 936

永久

修改注册表
在路径HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Command Processor下

添加一个字符串值，名为autorun，值为chcp 65001，即自动运行命令

永久修改Vscode的Console编码

在settings.json中添加键"terminal.integrated.shellArgs.windows"

若默认powershell的话则值为["-NoExit","chcp 65001"]，若为cmd则值为["/K chcp 65001 >nul"]

Error

cmd一直显示“系统找不到可用的路径”，原来是HKEY_CURRENT_USER\Software\Microsoft\Command Processor下有多余的autorun代码

powershell的启动运行命令写在C:\Users\<USERNAME>\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1中
